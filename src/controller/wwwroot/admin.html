⚠️ Deployment Notice: Multi-Bridge Provisioning

Adding or updating a bridge in this registry writes its metadata to config.yaml, but does not automatically provision or start the matching Docker container.

For each registered bridge to go online and accept AirPlay connections, a corresponding Docker service must be defined and running in your Docker deployment stack.

🛠️ Docker Compose Service Template

Paste the following service template block into your docker-compose.yml file under the services: section. Remember to customize the placeholders for each bridge you deploy:

  bridge-YOUR-BRIDGE-ID:
    image: ghcr.io/kiwiprojekt/recastify:main
    container_name: bridge-YOUR-BRIDGE-ID
    restart: unless-stopped
    depends_on:
      - icecast
      - mqtt
    cap_add:
      - SYS_NICE
    networks:
      airplay:
        ipv4_address: YOUR-CONTAINER-IP
    volumes:
      - ./config/shairport-sync.conf.template:/opt/config/shairport-sync.conf.template:ro
    environment:
      AIRPLAY_NAME: "YOUR-AIRPLAY-NAME"
      ICECAST_HOST: "icecast"
      ICECAST_PORT: "8000"
      ICECAST_SOURCE_PASSWORD: "YOUR-ICECAST-SOURCE-PASSWORD"
      ICECAST_MOUNT: "YOUR-ICECAST-MOUNT"
      AUDIO_BITRATE: "YOUR-BITRATE"
      BRIDGE_ID: "YOUR-BRIDGE-ID"
      CONTROLLER_URL: "controller:3000"
      MQTT_HOST: "mqtt"

After updating your compose stack file, deploy using docker compose up -d to launch the new AirPlay receiver container.
