⚠️ Deployment Notice: Multi-Bridge Provisioning

Registering a bridge in this UI writes its parameters to config.yaml, but does not automatically launch or configure the matching Docker container.

For each registered bridge to go online and advertise over AirPlay, a corresponding Docker service definition must be active in your deployment stack.

🛠️ Docker Compose Service Template

Insert the following service blocks under the services: section of your docker-compose.yml file. Adjust environment parameters to match each bridge definition:

  bridge-YOUR-BRIDGE-ID:
    image: ghcr.io/kiwiprojekt/recastify:main
    container_name: bridge-YOUR-BRIDGE-ID
    restart: unless-stopped
    depends_on:
      - icecast
      - mqtt
    cap_add:
      - SYS_NICE
    networks:
      airplay:
        ipv4_address: YOUR-CONTAINER-IP
    volumes:
      - ./config/shairport-sync.conf.template:/opt/config/shairport-sync.conf.template:ro
    environment:
      AIRPLAY_NAME: "YOUR-AIRPLAY-NAME"
      ICECAST_HOST: "icecast"
      ICECAST_PORT: "8000"
      ICECAST_SOURCE_PASSWORD: "YOUR-ICECAST-SOURCE-PASSWORD"
      ICECAST_MOUNT: "YOUR-ICECAST-MOUNT"
      AUDIO_BITRATE: "YOUR-BITRATE"
      BRIDGE_ID: "YOUR-BRIDGE-ID"
      CONTROLLER_URL: "controller:3000"
      MQTT_HOST: "mqtt"

After updating your compose file, deploy the changes using docker compose up -d to provision the receiver containers.
